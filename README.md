# Contacts REST API (v1.2) 📇

Фінальна версія REST API для зберігання та управління контактами.

## 🚀 Нові функції (ДЗ 13)

- **Документація (Sphinx):** До кожної функції та класу додано `docstrings`. Налаштовано генерацію HTML-документації через Sphinx.
- **Тестування (Pytest):** Написано інтеграційні та модульні тести (папка `tests/`). Покриття коду тестами перевищує **75%** (перевірено через `pytest-cov`). Використовується ізольована in-memory база SQLite.
- **Кешування (Redis):** Функція `get_current_user` тепер кешує дані користувача у Redis на 15 хвилин, що зменшує навантаження на PostgreSQL.
- **Скидання пароля:** Додано ендпоінти `/auth/forgot-password` та `/auth/reset-password` з відправкою JWT-токена (діє 1 годину) на email.
- **Ролі користувачів (RBAC):** Додано ролі `user` та `admin`. Ендпоінт оновлення аватара (`/users/avatar`) захищений залежністю `RoleChecker` і доступний лише для адміністраторів.
- **Refresh Tokens (Додатково):** Реалізовано пару токенів (`access_token` та `refresh_token`). Додано ендпоінт `/auth/refresh` для оновлення сесії.

## 📂 Структура проєкту

- `app/` — основний код (роутери, моделі, схеми, сервіси).
- `docs/` — конфігурація Sphinx для генерації документації.
- `tests/` — конфігурація Pytest та тестові файли.
